리플렉션(Reflection)은 프로그램 실행 중에 클래스의 정보를 검사하고 조작할 수 있는 기능을 제공하는 Java API입니다.
리플렉션을 사용하면 런타임 시점에 동적으로 클래스의 구조, 필드, 메소드, 생성자 등을 조작하고 실행할 수 있습니다.

주요 사용 사례는 다음과 같습니다.
1. 런타임 시점(프로그램 실행 중)에 클래스의 이름, 필드, 메서드, 어노테이션등에 대한 정보를 동적으로 얻을 수 있다.
2. 클래스의 생성자를 호출하여 객체를 생성하거나, 메서드를 호출하여 객체의 동작을 실행할 수 있다.
3. 객체의 필드 값을 읽거나 수정할 수 있다.
4. 클래스나 메서드에 적용된 어노테이션 정보를 동적으로 확인하고 처리할 수 있다.

단점은 다음과 같습니다.
1. 컴파일 시간에는 알 수 없는 정보를 찾고 호출하기 때문에 일반적인 메서드 호출보다 성능이 떨어질 수 있다.
2. private 필드나 메서드에 접근할 수 있기 때문에 캡슐화(보안)를 깨뜨릴 수 있다.
3. 코드의 가독성과 유지보수성이 저하될 수 있다. 정적 타입 검사를 받지 않기 때문에 컴파일러가 코드를 검증할 수 없다.
